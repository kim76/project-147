# Upgrades And Abilities

## Intent

Upgrades should create interesting choices, not just bigger numbers.

Players should feel different builds emerging over time.

## Tower Upgrade Categories

- damage;
- fire rate;
- range;
- crit chance;
- crit multiplier;
- armour piercing;
- area size;
- slow strength;
- status duration;
- ability cooldown;
- chain or splash behaviour.

## Alien Upgrade Categories

- health;
- speed;
- dodge;
- resistance;
- shield;
- regeneration;
- stun resistance;
- slow resistance;
- swarm bonus;
- tower-disruption effects.

## Special Abilities

Abilities should be readable, limited and fun to time.

Defence examples:

- overcharge a tower;
- freeze a path area;
- call a strike;
- deploy a decoy;
- repair the base once.

Alien examples:

- shield pulse;
- short tunnel leap;
- tower jammer;
- decoy swarm;
- rage burst after losses.

## Design Rules

- Every upgrade must have a clear gameplay purpose.
- Every special ability must have a counter or trade-off.
- Upgrade values should be data-driven.
- Shared formulas must live in one place.
- Random effects must use controlled randomness so they can be tested.
