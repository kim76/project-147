# Originality

## Purpose

Project 147 is an original tower-defence game. It can learn from the genre, but it must not become a clone of any existing title.

This document protects the project from drifting into copycat design while still allowing useful genre conventions.

## Inspiration Boundary

Allowed inspiration:

- grid or open-field tower placement;
- enemies travelling from spawn to objective;
- towers with range, damage, fire rate and upgrades;
- waves, bosses and escalating difficulty;
- readable mobile-friendly tactical play;
- optional ads and in-app purchases where appropriate.

Not allowed:

- copying another game's name, logo, store copy or marketing language;
- copying another game's characters, animals, aliens, towers or silhouettes;
- copying maps, layouts, wave sequences or campaign structure;
- copying UI layout, icon style, sound identity or visual effects closely;
- copying jokes, mission text, enemy names or upgrade names;
- recreating another game so closely that a player would call it the same game with new assets.

## Project Identity

Project 147 is an asymmetric sci-fi tower-defence game about a messy frontier conflict.

The player can play both sides:

- defence mode: protect unstable frontier infrastructure using towers, traps and improvised technology;
- alien mode: evolve and command alien squads to break through defensive layouts.

The core hook is not only building a defence. It is learning how defensive systems work from both sides, then exploiting that knowledge.

## Distinctive Pillars

### Two-Sided Play

The player can defend or attack.

This should affect progression, upgrades, level design and humour. Alien mode must not feel like a thin bonus mode.

### Improvised Frontier Tech

Towers should feel cobbled together, unstable and funny, but still readable.

Examples:

- railgun made from mining equipment;
- gravity pylon with warning lights;
- foam sprayer that slows aliens;
- drone kennel that launches angry repair drones;
- questionable reactor overcharge device.

### Weird Evolving Aliens

Aliens should feel like odd biological problem-solvers, not generic monsters.

Examples:

- shield carrier that protects nearby squads;
- burster that splits into smaller aliens;
- slick runner with high dodge;
- stoneback with armour resistance;
- jammer that disrupts tower fire rate.

### Readable Comedy

The game should be funny through animation, names, timing, reactions and consequences.

Gameplay readability comes first. Jokes must not obscure important combat information.

## Banned Similarity Checklist

Before adding a tower, alien, map, UI element, icon, mission or store asset, check:

- Would this be recognisable as another game's specific thing?
- Does it use the same protected theme or character relationship?
- Does it copy a visual silhouette too closely?
- Does it copy a map shape or level gimmick too closely?
- Does it use similar naming, jokes or upgrade language?
- Could it be described as a reskin rather than original work?

If the answer is yes or uncertain, redesign it.

## Current Working Premise

Frontier outposts are being raided by alien swarms because both sides are fighting over unstable energy junk no one fully understands.

The defence side is underfunded, overconfident and constantly bolting dangerous devices together.

The alien side is adaptive, petty and weirdly organised.

Both sides should be playable, upgradeable and entertaining.
