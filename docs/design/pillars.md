# Game Pillars

## 1. Strategy From Both Sides

The player should understand each battle from the defender and attacker perspective.

Defence mode asks: how do I build a layout that survives pressure?

Alien mode asks: how do I exploit that layout with the right squad, traits and timing?

## 2. Fast Readable Battles

The game should be quick to understand on a phone.

Important information must be visible without pausing:

- tower range;
- alien path;
- damage type;
- resistance interaction;
- active abilities;
- base danger;
- upgrade availability.

## 3. Meaningful Upgrades

Upgrades should change decisions, not merely inflate numbers.

Examples:

- trade fire rate for armour piercing;
- add splash at the cost of single-target damage;
- make aliens slower but resistant;
- give a squad dodge but reduce health;
- unlock an active ability that requires timing.

## 4. Funny But Tactical

Humour should make the game memorable without making it messy.

Good humour:

- silly tower names;
- alien reactions;
- failed experiment animations;
- ability descriptions with personality;
- result screen flavour.

Bad humour:

- blocking the screen;
- slowing repeated play;
- hiding mechanics;
- making upgrades unclear.

## 5. Small Changes Stay Safe

The project should support quick iteration through tests, data definitions and clean boundaries.

Changing a tower stat should not require editing a scene, a prefab, a formula and a UI string separately.
